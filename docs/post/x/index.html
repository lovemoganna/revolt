X resources [100%]

Xresources is a user-level configuration dotfile, typically located at ~/.Xresources. It can be used to set X resources, which are configuration parameters for X client applications. 1

Load resource file

Resources are stored in X server,so have to only be read once.They are also accessible to remote X11 clinets.(such as SSH 转发的内容).

  • .Xresources: replacing any current settings.

  • Could use command: echo "xrdb ~/.Xresources" xrdb ~/.Xresources

  • Load a resource file merger with the current settings: echo "xrdb -merge ~/.Xresources" xrdb -merge ~/.Xresources

Most Display manager load the /.Xresource file on login.

xinitrc

if you are using a copy of the default xinitrc as your .xinitrc it already merges ~/.Xresources.

If you are using a custom .xinitrc add the following line:

1

[[ -f ~/.Xresources ]] && xrdb -merge -I$HOME ~/.Xresources

Xresource syntax

  • Basic Syntax

    The syntax of an Xresource file is as follows:

    1

    name.Class.resource: value

    and here is a real world example:

    1

    xscreensaver.Dialog.headingFont: -*-fixed-bold-r-*-*-*-100-*-*-*-*-iso8859-1

  • Wildcard matched

    Using the previous example, if you want to apply the same font to all programs (not just XScreenSaver) that contain the class name Dialog which contains the resource name headingFont, you could write:

    1

    ?.Dialog.headingFont:     -*-fixed-bold-r-*-*-*-100-*-*-*-*-iso8859-1

    if you want to apply this same rule to all programs that contain the resource headingFont, regardless of its class, you could write:

    1

    *headingFont:    -*-fixed-bold-r-*-*-*-100-*-*-*-*-iso8859-1

  • Comments

    1

    ! This is a comment form

  • Inlcude files

    TO use different files for each application,use #include in the main file ,e.g. ~/.Xresources:

    1
2
3
4
5

    ~/.Xresources
   
--------------------------------------------------
   
#include ".Xresources.d/xterm"
#include ".Xresources.d/rxvt-unicode"
#include ".Xresources.d/fonts"
#include ".Xresources.d/xscreensaver"

    If file fail to load,specify the directory to xedb with the -I parameter.e.g.:

    1
2
3

    ~/.xinitrc
   
xrdb -I$HOME ~/.Xresources

  • Getting resource values

    if you want to get the value of a resource,you can use xgetres.e.g.:

    1

    xgetres xterm.metaSendsEscape
